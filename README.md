# Prompt Keeper

一个 [SillyTavern](https://github.com/SillyTavern/SillyTavern) 插件，用于按聊天会话保存和恢复 Chat Completion Prompt Manager 条目状态、排序、当前活跃预设，并在 v3.0.0 起提供世界书保护能力。

## v3.0.0 相比 v2.1.0 的更新重点

v2.1.0 主要解决免刷新热重载、按钮触控兼容、槽位弹窗展示和角色卡重导入后的 Prompt 排序恢复。v3.0.0 在保留这些能力的基础上，新增独立的世界书保护模块，并增强自动恢复稳定性。

### 新增：世界书保护

- 可按聊天保存当前启用的世界书列表。
- 可保存和恢复世界书条目的蓝灯/选择性状态、绿灯/常驻状态、禁用状态和条目顺序。
- 世界书条目顺序默认纳入保存和恢复范围，适合需要固定优先级的世界书配置。
- 支持多个世界书槽位，默认槽位会随保存更新。
- 支持手动保存、手动恢复、删除槽位，也可选择切换聊天时自动恢复世界书。

### 新增：世界书启用列表恢复策略

| 策略 | 说明 |
|------|------|
| 合并当前与保存列表（推荐） | 启用保存槽位里的世界书，同时保留酒馆当前已经启用的其它世界书 |
| 严格替换为保存列表 | 将当前启用世界书列表完全恢复为保存时的列表 |
| 不修改启用列表，只恢复条目 | 不改当前启用列表，只恢复能匹配到的世界书条目状态 |

### 改进：世界书条目匹配和安全恢复

- 优先按 `uid` / `id` 匹配世界书条目。
- 如果只能按名称匹配，会结合关键词、备注或内容摘要辅助判断。
- 如果无法唯一确认目标条目，会跳过该条目，降低误恢复风险。

### 改进：自动恢复稳定性

- 延长预设切换和 Prompt 状态就绪等待时间，减少聊天刚切换完成时恢复过早的问题。
- 自动恢复失败时会在同一聊天内有限重试，降低前端数据尚未加载完成导致的恢复失败概率。
- 热重载时会同时清理 Prompt Keeper 主模块和世界书模块的旧事件、旧 UI 与旧定时器。

### 改进：UI 与交互

- 世界书 UI 使用 SillyTavern 原生按钮类和主题变量，尽量跟随酒馆当前主题。
- 保存、恢复、删除按钮已绑定鼠标、触控和键盘 Enter / Space 事件。
- 设置页说明面向用户描述功能效果，不展示内部实现细节。

## 功能特性

- **按聊天独立配置**：保存和恢复 Prompt 条目启用/禁用状态、排列顺序和当前预设。
- **多槽位保存**：可选择保存时自定义槽位名称，同一聊天最多保存 5 个 Prompt 配置槽位。
- **自动恢复**：切换聊天时可自动恢复已保存配置，并带有防抖、脏检查和有限重试。
- **状态指示**：在操作栏显示当前聊天是否已有保存配置。
- **世界书保护**：可保存和恢复当前启用世界书列表、条目蓝灯/绿灯/禁用状态和条目顺序。
- **免刷新热重载**：更新插件文件且扩展入口被重新执行时会自动加载新版代码，也可通过 `window.PromptKeeperReload()` 手动热重载。

## 安装

### 通过 SillyTavern 扩展安装器

1. 打开 SillyTavern。
2. 进入 **扩展面板** → **安装扩展**。
3. 粘贴仓库地址：
   ```text
   https://github.com/sisjzknxhnsnejxn-cmyk/prompt-keeper
   ```
4. 点击安装。
5. 首次启用后建议刷新一次浏览器页面；后续更新通常可通过扩展入口自动热重载，或在控制台执行 `window.PromptKeeperReload()`。

### 手动安装

```bash
cd /path/to/SillyTavern/data/<user-handle>/extensions
git clone https://github.com/sisjzknxhnsnejxn-cmyk/prompt-keeper
```

首次安装后刷新浏览器页面。

## 使用方法

### Prompt 配置按钮

| 按钮 | 功能 |
|------|------|
| 保存 | 保存当前 Prompt 条目的启用状态、排序和当前预设名称 |
| 恢复 | 手动恢复已保存的 Prompt 配置，包括切换预设 |
| 删除 | 删除当前聊天的 Prompt 保存槽位 |

### Prompt 设置

| 选项 | 说明 |
|------|------|
| 启用插件 | 控制 Prompt Keeper 主功能开关 |
| 切换聊天时自动恢复 | 开启后切换聊天会自动恢复 Prompt 配置 |
| 保存时允许自定义名称 | 开启后保存时可输入槽位名称 |

### 世界书保护按钮

插件会在世界书相关区域加入世界书保护操作栏。

| 按钮 | 功能 |
|------|------|
| 保存世界书 | 保存当前世界书配置到当前聊天元数据 |
| 恢复世界书 | 从当前聊天的世界书槽位中选择并恢复 |
| 删除 | 删除当前聊天的世界书保存槽位 |

### 世界书保护设置

| 选项 | 说明 |
|------|------|
| 启用世界书保护 | 控制世界书保护模块开关 |
| 切换聊天时自动恢复世界书 | 开启后切换聊天会自动恢复世界书配置 |
| 保存/恢复已启用的世界书列表 | 是否在恢复时管理当前聊天启用的世界书列表；关闭后不改启用列表，只恢复能匹配到的条目 |
| 保存全部已知世界书条目 | 控制保存条目状态的范围；默认关闭时只保存当前启用世界书，开启后保存插件能识别到的全部世界书 |
| 恢复启用世界书列表 | 选择合并、严格替换或只恢复条目的策略 |
| 保存时允许自定义槽位名称 | 开启后可为世界书保存槽位命名 |

世界书条目的蓝灯/选择性状态、绿灯/常驻状态、禁用状态和条目顺序会始终保存并参与恢复，无需额外开启。

## 数据保存位置

Prompt 配置保存在当前聊天的：

```text
chatMetadata.promptKeeperState
```

世界书保护配置保存在当前聊天的：

```text
chatMetadata.promptKeeperWorldBookState
```

这些数据跟随聊天文件保存。导出或迁移聊天时，只要聊天元数据完整保留，配置也会随之保留。

## 管理范围

Prompt Keeper 管理 Prompt 条目状态、排序、当前 Chat Completion 预设名称、保存槽位、当前启用世界书列表，以及世界书条目的蓝灯/绿灯/禁用状态和排序。

Prompt Keeper 不管理模型选择、API 设置、生成参数、Instruct 预设、上下文模板或 SillyTavern 核心文件。

## 系统要求

- SillyTavern 1.12.0 或更高版本。
- 需要 SillyTavern 前端能正常访问 Prompt Manager 和世界书相关数据。

## 常见问题

**Q: 数据保存在哪里？**
A: 保存在当前聊天的 `chatMetadata.promptKeeperState` 和 `chatMetadata.promptKeeperWorldBookState` 中，跟随聊天文件一起存储。

**Q: 可以只使用 Prompt 保存，不使用世界书保护吗？**
A: 可以。在扩展设置中关闭「启用世界书保护」即可。

**Q: 世界书恢复会覆盖我当前临时启用的世界书吗？**
A: 默认策略是“合并当前与保存列表”，不会移除当前已经启用的其它世界书。如果选择“严格替换为保存列表”，则会覆盖当前启用列表。

**Q: 为什么有些世界书条目没有被恢复？**
A: 如果保存时的条目在当前世界书中不存在，或出现重名且无法唯一确认，插件会跳过该条目，以降低误改风险。

**Q: 更新后需要重启 SillyTavern 后端吗？**
A: 通常不需要。首次安装建议刷新浏览器页面；更新后如果扩展入口重新执行会自动热重载，也可以在控制台执行 `window.PromptKeeperReload()`。
